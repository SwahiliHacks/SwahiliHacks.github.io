# public/images/

These files are referenced throughout the site content but were not part of
the files provided for this rebuild — copy them in from your existing
`images/` folder before deploying, using these exact filenames:

| Filename | Used for |
|---|---|
| `profile.jpg.png` | Profile photo in the site header (every page) |
| `sh4.jpg` | Image in the site footer (every page) |
| `favicon.png` | Browser tab icon |
| `tool1.jpg` | Trading & Finance homepage card background |
| `tool2.jpg` | Productivity Suite homepage card background |
| `tool3.jpg` | Online Income homepage card background |
| `tool4.jpg` | Trading Automation homepage card background |
| `tool5.jpg` | Professional Courses homepage card background |
| `tool6.jpg` | Affiliate Platforms homepage card background |
| `tool7.jpg` | AI Tools homepage card background |
| `tool8.jpg` | Content Creation homepage card background |
| `tool9.jpg` | E-commerce homepage card background |
| `tool10.jpg` | Finance Management homepage card background |
| `tool11.jpg` | Remote Work homepage card background |
| `tool12.jpg` | Digital Services homepage card background |

Until these are added, the `<img>` tags and CSS background-images
referencing them will show broken-image icons / blank backgrounds — the
rest of the site (layout, links, text) works fine either way.

See `docs/CUSTOMIZATION.md` for how to add or swap images.
