# Installation Guide

Getting this repo running on a fresh computer, from zero.

## 1. Install Node.js

You need Node.js **22.12 or newer** (check `package.json` → `engines`).

- Download from https://nodejs.org (choose the LTS version, which is 22.x as of 2026)
- Verify it worked:
  ```bash
  node --version   # should print v22.x.x or higher
  npm --version    # should print 10.x or higher
  ```

npm (Node Package Manager) is installed automatically alongside Node — you
don't install it separately.

## 2. Get the code

If you already have Git installed:

```bash
git clone https://github.com/SwahiliHacks/SwahiliHacks.github.io.git
cd SwahiliHacks.github.io
```

If you don't have Git yet, install it from https://git-scm.com, or download
the repo as a ZIP from GitHub (green "Code" button → "Download ZIP") and
unzip it.

## 3. Install dependencies

From inside the project folder:

```bash
npm install
```

This reads `package.json`, downloads every library the project depends on
(Astro, Tailwind, the sitemap generator, etc.) into a `node_modules/`
folder. This folder is never committed to Git (see `.gitignore`) — it's
regenerated by `npm install` on any machine.

This step takes 30–90 seconds depending on your connection, and only needs
to be re-run when `package.json` changes (e.g. after `git pull` brings in a
new dependency).

## 4. Run it locally

```bash
npm run dev
```

Open http://localhost:4321 in your browser. Edit any file in `src/` and the
page updates automatically — no manual refresh, no build step, while
developing.

## 5. Build the production version (optional, to test before deploying)

```bash
npm run build      # outputs static HTML/CSS/JS into dist/
npm run preview    # serves dist/ locally so you can check the real build
```

## Troubleshooting

If something goes wrong at any of these steps, see
[TROUBLESHOOTING.md](TROUBLESHOOTING.md) before asking for help — the most
common issues (wrong Node version, stale `node_modules`) are covered there.
